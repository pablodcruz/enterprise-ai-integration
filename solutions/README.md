## Comparison Checklist

After opening a solution, compare:

- Component and trust boundaries
- Tool and data schemas
- Authorization enforcement
- Failure handling
- Observability and audit evidence
- Test strategy
- Evaluation coverage
- Setup and reset experience

Record which differences in your implementation are improvements, tradeoffs, omissions, or mistakes.
