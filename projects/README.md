## Requirement Document Convention

Every project specification contains:

- Problem and audience
- Learning outcomes
- Functional requirements
- Security and reliability requirements
- Required failure scenarios
- Minimum architecture
- Deliverables
- Acceptance criteria
- Evaluation rubric
- Stretch goals

Requirements describe observable behavior. They avoid prescribing every class, framework, or internal implementation choice unless interoperability depends on it.
